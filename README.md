# 🕌 Quran Broadcast

Real-time, fullscreen (1920×1080) Quran broadcast web app for OBS Browser Source.

## Architecture

```
quran/
├── scripts/ingest.mjs      ← Ingestion script (run once to build manifest)
├── server/index.mjs        ← Express API server (port 3737)
├── client/                 ← React + TypeScript + Vite frontend (port 5173)
│   └── src/
│       ├── App.tsx                     ← Playback state machine
│       ├── components/
│       │   ├── BroadcastScene.tsx      ← 1920×1080 SVG canvas compositor
│       │   ├── TopWindow.tsx           ← Slideshow (Ken Burns + crossfade)
│       │   ├── InfoBand.tsx            ← Reciter / Surah / Juz / Page band
│       │   ├── QuranWindow.tsx         ← Page scroll + content-bounds crop
│       │   └── ControlsPanel.tsx       ← Play/pause/jump/settings UI
│       ├── hooks/
│       │   ├── useManifest.ts          ← Fetches manifest + config + slides
│       │   ├── useAudio.ts             ← Audio element management
│       │   └── useSlideshow.ts         ← Slideshow cycle state
│       └── utils/
│           └── contentBounds.ts        ← Canvas alpha-scan crop detection
├── public/manifest.json    ← Generated by ingest script
├── config.json             ← User-editable settings
└── package.json
```

## Quick Start

### Step 1 — Install dependencies
```powershell
cd "D:\2025 apps\quran"
npm install
cd client; npm install; cd ..
```

### Step 2 — Run ingestion (first time or when adding pages)
Copy `config.example.json` to `config.json` and adjust local paths before running ingest/server.

```powershell
npm run ingest
```
Reads all pages from `hafs/`, `mp3/`, `slide/`, extracts audio durations via `music-metadata`, and writes `public/manifest.json`.

### Step 3 — Start servers
**Terminal 1 — Express server:**
```powershell
npm run server
```

**Terminal 2 — Vite frontend:**
```powershell
cd client; npm run dev
```

Open: **http://localhost:5173**

### OBS Browser Source
After building the client (`cd client; npm run build`), add an OBS Browser Source pointing to:
```
http://localhost:3737
```
Width: 1920, Height: 1080, Custom CSS: `body { margin: 0; overflow: hidden; }`

## Configuration

Edit `config.json`:

| Field | Default | Description |
|---|---|---|
| `reciterName` | `شيخ خالد الجليل` | Shown in info band |
| `startPage` | `1` | Which page to start from |
| `loopMode` | `true` | Loop after last page |
| `slideshowInterval` | `8000` | Ms between slide changes |
| `slideshowTransitionDuration` | `1500` | Ms for crossfade |
| `scrollZoomFactor` | `1.0` | Scale factor for page (increase to show fewer lines) |
| `pageTransitionDuration` | `800` | Ms for page transition |
| `hafsDir` | *(your path)* | Folder with `.webp` + `.json` page files |
| `mp3Dir` | *(your path)* | Folder with `Page001.mp3` audio files |
| `slideDir` | *(your path)* | Folder with slideshow images |

## Debug Mode

Press `D` or click 🐛 Debug in controls to show:
- **Red dashed rect** — detected content bounds (alpha-scanned crop box)
- **Cyan polygons** — ayah bounding boxes from per-page JSON

## Layout (1920×1080)

```
┌────────────────────────────────┐  ← y=0
│       TOP SLIDESHOW            │    370px — Ken Burns + crossfade
│       (SVG clipPath masked)    │
├────────────────────────────────┤  ← y=370
│  Reciter  │  سورة …  │ Juz/Pg │    74px — info band
├────────────────────────────────┤  ← y=444
│       QURAN PAGE SCROLL        │    636px — content-bounds scroll
│       (SVG clipPath masked)    │          from bottom to top over audio duration
└────────────────────────────────┘  ← y=1080
```

Each region uses an **independent SVG `<clipPath>`** so content can never bleed between zones.

## Manifest Format

`public/manifest.json` — array of 604 entries:
```json
{
  "page": 3,
  "imagePath": "/assets/hafs/003.webp",
  "jsonPath": "/assets/hafs/003.json",
  "audioPath": "/assets/mp3/Page003.mp3",
  "audioDuration": 47.2,
  "surah": { "id": 2, "nameArabic": "البقرة", "nameSimple": "Al-Baqarah" },
  "juz": 1
}
```

## Future Extension: Ayah-Level Sync

Per-page JSON files (`hafs/NNN.json`) contain:
```json
[{
  "ayahNumber": 6, "surahNumber": 2,
  "polygon": "343,10 0,10 0,46.11 ...",
  "page_number": 3, "juz": 1
}]
```

To add ayah-level highlighting in a future phase:
1. Build a per-page word timing table (e.g., from Tarteel/EveryAyah APIs or embedded CUE data)
2. In `QuranWindow.tsx`, listen to `audio.currentTime` on each RAF tick
3. Look up which ayah polygon is active and render a highlight overlay
4. The `detectContentBounds()` scale factor is already available to map polygon px → screen px

The `contentBounds.ts` cache + scale infrastructure is already in place for this.
