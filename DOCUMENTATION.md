# توثيق مشروع بث القرآن الكريم

## نظرة عامة

مشروع "بث القرآن الكريم" هو تطبيق ويب يتيح بث صفحات القرآن الكريم بشكل مباشر مع مزامنة الصوت والصورة، ويُستخدم كمصدر متصفح في OBS لتقديم بث احترافي. يتضمن التطبيق واجهة عرض للصفحات مع ميزة التمرير التدريجي للنص أثناء تشغيل التلاوة الصوتية.

## هيكل المشروع

```
quran/
├── scripts/ingest.mjs      ← سكربت الاستيعاب (يتم تشغيله مرة واحدة لإنشاء الملف التعريفي)
├── server/index.mjs        ← خادم Express API (المنفذ 3737)
├── client/                 ← واجهة المستخدم باستخدام React + TypeScript + Vite (المنفذ 5173)
│   └── src/
│       ├── App.tsx                     ← آلة حالة التشغيل
│       ├── components/
│       │   ├── BroadcastScene.tsx      ← مركبة كانفاس SVG بحجم 1920×1080
│       │   ├── TopWindow.tsx           ← عرض الشرائح (تأثير كين بيرنز + تلاشي)
│       │   ├── InfoBand.tsx            ← شريط المعلومات (القارئ / السورة / الجزء / الصفحة)
│       │   ├── QuranWindow.tsx         ← تمرير الصفحة + قص حدود المحتوى
│       │   └── ControlsPanel.tsx       ← واجهة تشغيل/إيقاف/ nhbr/الإعدادات
│       ├── hooks/
│       │   ├── useManifest.ts          ← جلب الملف التعريفي + الإعدادات + الشرائح
│       │   ├── useAudio.ts             ← إدارة عنصر الصوت
│       │   └── useSlideshow.ts         ← حالة دورة عرض الشرائح
│       └── utils/
│           └── contentBounds.ts        ← اكتشاف قص محتوى القناة ألفا
├── public/manifest.json    ← يتم إنشاؤه بواسطة سكربت الاستيعاب
├── config.json             ← إعدادات قابلة للتحرير من قبل المستخدم
└── package.json
```

## المكونات الرئيسية

### 1. خادم Express (server/index.mjs)
- يخدم الأصول الثابتة (hafs، mp3، slide) ونهايات API
- يوفر واجهات برمجة التطبيقات للحصول على الإعدادات والملفات التعريفية وقوائم الشرائح
- يدعم تحديث الإعدادات في الوقت الفعلي عبر طلب PATCH

### 2. واجهة العميل (client/src/App.tsx)
- يدير حالة التشغيل: الصفحة الحالية، التشغيل/الإيقاف، الوقت المنقضي
- يتعامل مع تحميل وتشغيل الصوت
- يحتوي على اختصارات لوحة المفاتيح

### 3. مشهد البث (BroadcastScene.tsx)
- مركبة SVG بحجم 1920×1080 تحتوي على عدة طبقات:
  - الخلفية
  - TopWindow (عرض الشرائح داخل فتحة القوس العلوية للإطار)
  - QuranWindow (تمرير صفحة القرآن داخل فتحة القوس السفلية للإطار)
  - frame.png (إطار تزييني يغطي/يقطع حواف المحتوى)
  - InfoBand (اسم السورة / الجزء / الصفحة يظهر فوق الإطار)

### 4. نافذة القرآن (QuranWindow.tsx)
- تمرير تدريجي للصفحات أثناء تشغيل التلاوة الصوتية
- يحسب حدود المحتوى القابلة للعرض ويؤدي تمريراً متناسقًا
- يدعم التبديل السلس بين الصفحات مع الاحتفاظ بتدفق النص

## الإعداد والاستخدام

### الخطوة 1 - تثبيت التبعيات
```powershell
cd "D:\2025 apps\quran"
npm install
cd client; npm install; cd ..
```

### الخطوة 2 - تشغيل الاستيعاب (في المرة الأولى أو عند إضافة صفحات)
```powershell
npm run ingest
```
يقرأ جميع الصفحات من `hafs/` و `mp3/` و `slide/`، ويستخرج مدة الصوت عبر `music-metadata`، ويكتب `public/manifest.json`.

### الخطوة 3 - بدء الخوادم
**نافذة 1 - خادم Express:**
```powershell
npm run server
```

**نافذة 2 - واجهة Vite:**
```powershell
cd client; npm run dev
```

افتح: **http://localhost:5173**

### استخدام OBS كمصدر متصفح
بعد بناء العميل (`cd client; npm run build`)، أضف مصدر متصفح في OBS يشير إلى:
```
http://localhost:3737
```
العرض: 1920، الارتفاع: 1080، CSS مخصص: `body { margin: 0; overflow: hidden; }`

## التكوين

يمكن تعديل `config.json`:

| الحقل | القيمة الافتراضية | الوصف |
|-------|-------------------|------|
| `reciterName` | `ماهر المعيقلي` | يظهر في شريط المعلومات |
| `startPage` | `1` | الصفحة التي يبدأ منها |
| `loopMode` | `true` | تكرار بعد آخر صفحة |
| `slideshowInterval` | `8000` | ميلي ثانية بين تغييرات الشرائح |
| `slideshowTransitionDuration` | `1500` | ميلي ثانية لتأثير التلاشي |
| `scrollZoomFactor` | `0.5` | عامل مقياس للصفحة (زده لعرض سطور أقل) |
| `pageTransitionDuration` | `800` | ميلي ثانية لانتقال الصفحة |
| `hafsDir` | `(مسارك)` | مجلد يحتوي على ملفات `.webp` و `.json` للصفحات |
| `mp3Dir` | `(مسارك)` | مجلد يحتوي على ملفات الصوت `Page001.mp3` |
| `slideDir` | `(مسارك)` | مجلد يحتوي على صور عرض الشرائح |

## وضع التصحيح

اضغط `D` أو انقر على "Debug" في عناصر التحكم لعرض:
- **مستطيل أحمر متقطع** — حدود المحتوى المكتشفة (مربع القص الممسوح ضوئيًا)
- **مضلعات زرقاء** — مربعات تحيط بالآيات من ملف JSON لكل صفحة

## التصميم (1920×1080)

```
┌────────────────────────────────┐  ← y=0
│       عرض الشرائح العلوي       │    370 بكسل — تأثير كين بيرنز + تلاشي
│       (تم قصه بمسار SVG)       │
├────────────────────────────────┤  ← y=370
│  القارئ  │  سورة ...  │ جزء/صفحة │    74 بكسل — شريط المعلومات
├────────────────────────────────┤  ← y=444
│       تمرير صفحة القرآن        │    636 بكسل — تمرير حدود المحتوى
│       (تم قصه بمسار SVG)       │          من الأسفل إلى الأعلى خلال مدة الصوت
└────────────────────────────────┘  ← y=1080
```

يستخدم كل منطقة **`<clipPath>` SVG مستقلة** بحيث لا يمكن أن يتسرب المحتوى بين المناطق.

## تنسيق الملف التعريفي

`public/manifest.json` — مصفوفة من 604 إدخال:
```json
{
  "page": 3,
  "imagePath": "/assets/hafs/003.webp",
  "jsonPath": "/assets/hafs/003.json",
  "audioPath": "/assets/mp3/Page003.mp3",
  "audioDuration": 47.2,
  "surah": { "id": 2, "nameArabic": "البقرة", "nameSimple": "Al-Baqarah" },
  "juz": 1
}
```

## التوسع المستقبلي: مزامنة مستوى الآية

تحتوي ملفات JSON لكل صفحة (`hafs/NNN.json`) على:
```json
[{
  "ayahNumber": 6, "surahNumber": 2,
  "polygon": "343,10 0,10 0,46.11 ...",
  "page_number": 3, "juz": 1
}]
```

لإضافة تمييز مستوى الآية في مرحلة مستقبلية:
1. بناء جدول توقيت كلمات لكل صفحة (مثلاً من واجهات برمجة تطبيقات Tarteel/EveryAyah أو بيانات CUE المضمنة)
2. في `QuranWindow.tsx`، الاستماع إلى `audio.currentTime` في كل دورة RAF
3. البحث عن مضلع الآية النشطة وتقديم تراكب تمييز
4. عامل المقياس `detectContentBounds()` متوفر بالفعل لرسم خرائط من بكسل مضلع → بكسل الشاشة

## ميزات إضافية

- دعم الوضع التلقائي لتقديم مقاطع فيديو (render mode)
- تأثيرات حركية اختيارية (parallax، gold sweep)
- دعم عرض شرائح خلفية
- واجهة تحكم قابلة للإخفاء
- دعم ملء الشاشة

## التقنيات المستخدمة

- React + TypeScript + Vite (واجهة العميل)
- Express.js (خادم API)
- SVG (الرسومات وقص المحتوى)
- Web Audio API (إدارة الصوت)
- Puppeteer (لإنشاء مقاطع فيديو عند الطلب)
- music-metadata (لقراءة معلومات الصوت)

## الملفات المهمة

- `scripts/ingest.mjs`: سكربت إنشاء الملف التعريفي
- `scripts/render.mjs`: سكربت إنشاء مقاطع الفيديو
- `hafs/*.json`: بيانات مواقع الآيات على الصفحات
- `hafs/*.webp`: صور الصفحات
- `mp3/*.mp3`: تسجيلات التلاوة
- `slide/*.jpeg`: صور خلفية عرض الشرائح
